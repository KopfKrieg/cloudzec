Gruppe hinzufügen:
 # groupadd [group]
  -> groupadd cloud


User hinzufügen:
 Der Switch -m erzeugt ein Homeverzeichnis in /home/[user]
 Die Login-Shell muss auf /bin/bash gesetzt werden, es sei denn, man will ein sftp-chroot-jail
 # useradd -g [initial_group] -G [additional_groups] -s [login_shell] [username]
  -> useradd -g cloud -G http -s /bin/nologin cloud


Homeverzeichnis erstellen:
 # mkdir -p [homedir]
  -> mkdir -p /home/chroot/cloud


Homeverzeichnis zuweisen:
 # usermod -d [homedir] [user]
  -> usermod -d /home/chroot/cloud cloud


Passwort ändern:
 # passwd [user]
  -> passwd cloud


/bin/nologin zu den erlaubten Shells hinzufügen:
 # nano /etc/shells
  -> /bin/nologin zur Liste hinzufügen


Für ein SFTP-Jail:
 In der Datei /etc/ssh/sshd_config das sftp-Subsystem auskommentieren (mit einem #) bzw.
 durch ein anderes ersetzen:

  Subsystem       sftp    internal-sftp

 Dann das chroot-jail erzeugen, entweder auf Basis der Gruppenzugehörigkeit (für mehrere User):

  Match Group [group]
    ChrootDirectory %h
    ForceCommand internal-sftp
    AllowTcpForwarding no

 Oder auf Basis des Benutzernamens (für einen oder eine handvoll Nutzer):

  Match User [user]
    ChrootDirectory %h
    ForceCommand internal-sftp

 %h entspricht dem Homeverzeichnis des Nutzers der sich anmeldet, theoretisch könnte man für 
 mehrere Nutzer also auch ein gemeinsames chroot-jail erzeugen.
 Des Weiteren muss der Besitzer des chroot-Verzeichnisses geändert werden:
  # chown root ~[user]
   -> chown root ~cloud

 Und der SSH-Dienst neu gestartet:
  # systemctl restart sshd.service
   -> systemctl restart sshd.service

 Das chroot-jail ist damit fertig und einsatzbereit. Zum Testen sollte man sich von einem
 anderen PC aus einloggen, ersteres muss fehlschlagen, zweiteres muss gelingen:
 $ ssh [user]@[host]
  -> ssh cloud@10.0.0.24

 $ sftp [user]@[host]
  -> sftp cloud@10.0.0.24

 Damit man Dateien übertragen kann, muss man noch ein Verzeichnis erstellen und diesem
 entsprechende Rechte geben:
 # mkdir -p [homedir]/[verzeichnis]
  -> mkdir -p /home/chroot/cloud/drop
